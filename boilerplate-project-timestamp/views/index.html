Timestamp Microservice

User Stories:

  1. I can pass a string as a parameter, and it will check if that string contains either a valid Unix timestamp or a valid date string (e.g. "2016-12-25")
  2. If it is valid, a JSON response will be returned with a unix key that is a Unix timestamp in milliseconds and a utc key that is a string of the input formatted as GMT date string
  3. If the input is invalid, the API returns a JSON object with an error key and value "Invalid Date"
  4. If no date parameter is provided, the current time will be used

Example Usage:

  • /api/2015-12-25
  • /api/1451001600000
  • /api/ (current time)

Example Output:

{"unix":1451001600000,"utc":"Fri, 25 Dec 2015 00:00:00 GMT"}
